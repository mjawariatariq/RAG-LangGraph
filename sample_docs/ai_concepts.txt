Transformers are a type of deep learning architecture introduced in the 2017 paper "Attention Is All You Need". They use self-attention mechanisms to process sequences of data in parallel, making them much faster to train than previous recurrent neural networks (RNNs). Transformers are the foundation of almost all modern large language models including GPT, Claude, and Gemini.

Retrieval-Augmented Generation (RAG) is a technique that combines a retrieval system with a language model. Instead of relying purely on the model's trained knowledge, RAG first searches a knowledge base for relevant documents, then passes those documents as context to the LLM when generating a response. This allows the LLM to answer questions about private or up-to-date information it wasn't trained on.

Embeddings are numerical vector representations of text (or images, audio, etc.) that capture semantic meaning. Two pieces of text with similar meaning will have embeddings that are close together in vector space. Embeddings are generated by encoder models and are the core technology behind semantic search and RAG systems.

Vector databases are specialized databases designed to store and efficiently search through high-dimensional embeddings. Given a query embedding, they can quickly find the most similar vectors using approximate nearest neighbor (ANN) algorithms. Popular vector databases include Pinecone, Weaviate, Chroma, and Qdrant. FAISS is a popular open-source library for vector search.

FAISS (Facebook AI Similarity Search) is an open-source library developed by Meta for efficient similarity search and clustering of dense vectors. It supports both CPU and GPU and is designed to handle millions to billions of vectors. FAISS is commonly used as a local vector store for RAG applications because it requires no external server and can be saved to disk.

LangGraph is a library built on top of LangChain for building stateful, multi-actor applications with LLMs. It models agent workflows as graphs where nodes are functions (like LLM calls or tool executions) and edges define the flow of control. LangGraph supports cycles, branching, and persistent state, making it ideal for complex agentic applications.

Large Language Models (LLMs) are neural networks with billions of parameters trained on massive text datasets to predict the next token in a sequence. Through this training, they develop the ability to understand and generate human-like text. Modern LLMs like GPT-4, Claude 3, and Gemini Ultra can perform reasoning, coding, analysis, and creative writing. They are the core component in most modern AI assistants.

Fine-tuning is the process of taking a pre-trained model and continuing to train it on a smaller, task-specific dataset. Fine-tuning adapts the model's weights to perform better on a particular domain or task. Techniques like LoRA (Low-Rank Adaptation) make fine-tuning much more efficient by only updating a small subset of parameters.

Prompt engineering is the practice of designing and optimizing the text inputs (prompts) given to a language model to get the best possible outputs. Techniques include few-shot prompting (providing examples in the prompt), chain-of-thought prompting (asking the model to reason step by step), and system prompts (setting the model's role and context). Good prompt engineering can dramatically improve LLM performance without any model training.

AI Agents are LLM-powered systems that can take actions in the world by calling tools, browsing the web, writing and executing code, and more. An agent uses an LLM as its reasoning engine to decide which actions to take in order to achieve a goal. The ReAct pattern (Reason + Act) is a popular framework where the LLM alternates between reasoning about the problem and taking actions. LangGraph is commonly used to build agent workflows.
